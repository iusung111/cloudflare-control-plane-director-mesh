# adr_001_event_driven_control

## Status
Accepted

## Decision
The system treats state changes as events and derives current state from the event log instead of overwriting state directly.

## Context
- Workers
- reviewers
- multiple runtime lanes
- strict deploy boundaries
- need for traceability and replay

## Consequences
- All writes map to events.
 - Current state is derived, not directly authoritative.
 - Dedup, conflict, and guardrail checks happen before side effects.
 - Recorded events are the basis for audit, review, and rollback.

## Alternatives considered
- Direct state overwrite: rejected because it hurts traceability and conflict recovery.
- Partial event logging: rejected because it splits source of truth.
